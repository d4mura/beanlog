# BeanLog リサーチレビュー & 戦略提案

**作成日:** 2026-02-12
**作成者:** Analyst
**対象:** Director (だーむら。)

---

## 1. Overview — 現状の整理

BeanLog = 「コーヒー豆のVivino」。スペシャルティコーヒー愛好家向けの豆レビュー・発見プラットフォーム（PWA）。

**Phase 0 MVP スコープ:** 豆DB 200件 / レビュー投稿 / 検索・フィルタ / 日英対応 / Supabase Auth
**技術スタック:** Next.js + FastAPI + Supabase (PostgreSQL + pgvector)

---

## 2. Signals — 市場シグナル

### 2.1 マクロ環境

| シグナル | 示唆 |
|---------|------|
| 日本コーヒー市場 約2.9兆円、スペシャルティ比率13.4%（拡大中） | スペシャルティはまだニッチだが成長軌道。狙うべきセグメント |
| サードウェーブ → フォースウェーブへの移行 | 消費者が「産地・農園・精製」まで気にする時代。BeanLogのデータ構造はここに刺さる |
| PostCoffee 会員9万人超・継続率98% | コーヒーサブスクは定着済み。「買う前に調べる」需要は未充足 |
| インバウンド回復 + 日本コーヒー文化の海外注目 | 多言語対応は正解。特にアジア圏（台湾・韓国・タイ）からの関心が高い |

### 2.2 ユーザーインサイト（SNS/コミュニティ観察）

| パターン | 頻度 | BeanLogへの示唆 |
|---------|------|----------------|
| Instagramストーリーズでラテアート/パッケージ投稿 → 24h消滅 | 非常に高い | **記録のストック化**が最大の刺さりポイント |
| X/Twitterで「この豆どこで買える？」質問 | 中程度 | **購入導線**が価値。Phase 1のアフィリエイト前でも外部リンクで対応可 |
| r/coffee, r/JapaneseFood で日本ロースター質問 | 中程度 | **英語対応は初日から必須**。Reddit/海外フォーラムでの認知獲得チャンス |
| 「豆のメモ帳」的な個人スプレッドシート運用 | 低〜中 | 既存の代替手段がショボい = 参入チャンス |

## 3. Competitor Moves — 競合の最新動向

### 3.1 直接競合

| プレイヤー | 最新動向 | 強み | 弱み | BeanLogとの差別化ポイント |
|-----------|---------|------|------|------------------------|
| **Beanconqueror** (独) | オープンソース、Bluetooth機器連携強化、抽出パラメータ特化 | 抽出記録の深さ、デバイス連携、無料 | ソーシャル機能なし、豆DBなし、UIが玄人向け | BeanLogは**発見+コミュニティ**。Beanconquerorは抽出オタク向け |
| **PostCoffee** (日) | ECモール化（CFFBNS展開）、海外ロースター取り込み（Onyx, ONA Coffee等） | 物販基盤、ロースターネットワーク、診断UX | UGCレビューが弱い、自社販売優先でフラットな比較不可 | BeanLogは**中立的UGCプラットフォーム**。販売者バイアスなし |
| **コーヒー豆研究所** (日) | SEOアフィリメディア継続 | SEO強い、記事量 | 広告寄り、ユーザー参加なし、データベース型ではない | BeanLogは**構造化データ + ユーザー参加型** |

### 3.2 間接競合・参考プレイヤー

| プレイヤー | モデル | 参考になる点 |
|-----------|--------|------------|
| **Vivino** (ワイン) | ラベルスキャン + UGC + EC連携 | MVP→スケールの成功パターン。ただし2023年以降リストラ。UGCの質維持が課題だった |
| **Untappd** (ビール) | チェックイン型レビュー + バッジ/ゲーミフィケーション | エンゲージメント設計が秀逸。BeanLogにバッジ機能は検討の価値あり |
| **Tabelog/食べログ** (飲食) | UGC + SEO + 有料送客 | ロースター課金モデルの先行事例。ただし「やらせ問題」は教訓 |

### 3.3 競合マップ上のポジション

```
        UGCレビュー充実
             ↑
             |
  Untappd ● |         ● BeanLog（目指す位置）
             |        /
  Vivino ●   |       /
             |      /
  ─────────────────────────→ コーヒー特化度
             |    /
  食べログ ●  |   /
             |  ● PostCoffee
             | ● コーヒー豆研究所
             |
        UGCレビュー弱い
```

**結論: 「コーヒー特化 × UGC充実」の象限はガラ空き。** Beanconquerorは抽出特化でソーシャルがない。PostCoffeeはEC特化でUGCが弱い。

| # | リスク | 影響度 | 発生確率 | 対策案 |

## 5. 提案 — Next Actions

### 提案①: データ戦略の前倒し・強化 【優先度: 最高】

**問題:** 豆DB 200件ではユーザーが「自分の豆が見つからない」→ 即離脱

**アクション:**
- **Phase 0 で 500件** を目標に引き上げ
- スクレイピング対象を拡大: PostCoffee掲載ロースター（現時点で30社以上確認）、SCAJ会員リスト、各ロースターEC
- データ収集パイプラインを半自動化（LLM + 人力チェック）
- **ユーザーによる豆登録機能**をMVPに追加検討（未登録豆のレビュー時に「この豆を登録」フロー）

### 提案②: コールドスタート対策（シードUGC） 【優先度: 最高】

**問題:** レビュー0件のプラットフォームに誰も書き込まない

**アクション:**
- ローンチ前に **チーム内で最低50件のレビュー** を投稿
- 提携ロースター5店舗の常連客 **各10名にベータ招待** → 初期50件のリアルレビュー
- **Instagramストーリーズからのインポート機能**（将来）の布石として、レビュー投稿UIに「写真」フィールドを追加

### 提案③: MVP スコープの微調整 【優先度: 高】

現在のMVP仕様を見て、以下を推奨:

| 追加すべき | 理由 |
|-----------|------|
| **レビュー時の写真投稿**（1枚） | Instagramユーザーの行動様式に合わせる。Supabase Storageは既にスタック内 |
| **豆の「飲みたい」リスト** | 発見→記録のフックになる。実装コスト低い |
| **SNSシェアボタン**（OGP対応） | Phase 1に回されているが、OGPはSSRで自然にできる。初期グロースに必須 |

| 削るor後回しにすべき | 理由 |
|-----------|------|
| **バーコードスキャン（E-08）** | 日本のコーヒー豆はJANコード未統一。実用性が低く、開発コストに見合わない |
| **フレーバーベクトル検索** | Phase 0では豆数が少なくレコメンドの精度が出ない。手動タグフィルタで十分 |

### 提案④: ロースター獲得戦略の具体化 【優先度: 高】

**アクション:**
- **ターゲットリスト作成:** 東京・大阪のスペシャルティロースター上位20社をリストアップ
- **提携メリットの明確化:** 無料掲載 + レビューデータフィードバック + EC導線
- **SCAJ 2026（秋）をマイルストーンに:** カンファレンス出展 or ブースで認知獲得
- **初期5店舗はDirectorの人脈 or コールドメール**で獲得

### 提案⑤: KPI の見直し 【優先度: 中】

| 現行KPI | 提案KPI | 理由 |
|---------|---------|------|
| 豆DB 200件 | **500件** | 発見体験の閾値 |
| 登録ユーザー 100人 | **100人**（据置） | 妥当 |
| レビュー 500件 | **200件** | 500件は2ヶ月で非現実的。質を重視 |
| 提携ロースター 5店舗 | **5店舗**（据置） | 妥当 |
| WAU 50人 | **30人** | 初期は少数でもリテンション率を見る |
| ー | **レビュー再訪率 40%**（新規追加） | UGCプラットフォームの生命線 |

### 提案⑥: Go-to-Market の初手 【優先度: 中】

1. **r/coffee, r/JapaneseFood に英語で紹介投稿** → 海外ユーザー獲得（コスト0）
2. **コーヒー系YouTuber/インフルエンサーにベータ版提供** → James Hoffmann的なチャンネルは日本のコーヒー文化に興味あり
3. **XでPostCoffeeの豆をレビューする投稿** → PostCoffeeユーザー層を引き込む
4. **ロースター店頭にQRコード設置**（提携店舗）→ オフライン→オンラインの導線

## 6. まとめ

**BeanLogのポジションは正しい。「コーヒー特化 × UGC」の空白地帯は確実に存在する。**

ただし、最大のリスクは**データ量とUGCのコールドスタート**。技術的な準備はできているが、ローンチ前の「仕込み」が勝敗を分ける。

**最優先アクション（今週やるべきこと）:**
1. 豆データ収集パイプラインの設計・着手（目標500件）
2. 提携候補ロースター20社のリストアップ
3. MVP仕様のスコープ微調整（写真投稿追加、バーコード後回し）
